# Подробная инструкция деплоя Next.js на Beget

## Шаг 1: Подготовка проекта (выполнено)

Создан файл `server.js` для запуска приложения на Node.js.
Добавлен скрипт `start:server` в package.json.

## Шаг 2: Подготовка файлов для деплоя

Создайте файл `.gitignore` в корне проекта (если нет) и добавьте:
```
node_modules
.next
.env.local
.DS_Store
```

## Шаг 3: Создание архива проекта

В локальном терминале выполните:

```bash
# В папке проекта
cd /home/aleks/projects/my-website

# Удалите папки node_modules и .next (если есть)
rm -rf node_modules .next

# Создайте архив проекта (без node_modules и .next)
tar -czf my-website.tar.gz --exclude='node_modules' --exclude='.next' --exclude='.git' .
```

## Шаг 4: Загрузка на Beget через панель управления

### 4.1 Вход в панель Beget
1. Зайдите в панель управления Beget (beget.com)
2. Войдите в свой аккаунт

### 4.2 Создание сайта (если нет)
1. Перейдите в раздел "Сайты"
2. Нажмите "Добавить сайт"
3. Выберите домен (или поддомен)
4. Выберите тип: "Без CMS"
5. Нажмите "Создать"

### 4.3 Включение Node.js
1. В списке сайтов нажмите на ваш домен
2. Перейдите в "Настройки" → "Node.js"
3. Нажмите "Включить Node.js"
4. Выберите версию Node.js: 18.x или 20.x
5. Укажите путь к папке: `public_html`
6. В поле "Файл запуска" введите: `server.js`
7. Нажмите "Сохранить"

### 4.4 Загрузка файлов через файловый менеджер
1. В панели управления перейдите в "Файловый менеджер"
2. Откройте папку `public_html`
3. Удалите все файлы (если есть)
4. Нажмите "Загрузить"
5. Выберите созданный архив `my-website.tar.gz`
6. После загрузки нажмите на архив и выберите "Распаковать"

## Шаг 5: Установка зависимостей и сборка на сервере

### 5.1 Подключение по SSH
1. В панели управления перейдите в "SSH"
2. Найдите данные для подключения (хост, порт, логин, пароль)
3. Подключитесь по SSH (через терминал или PuTTY на Windows):

```bash
ssh username@server.beget.com
```

### 5.2 Переход в папку сайта
```bash
cd /home/username/public_html
```

### 5.3 Установка Node.js (если не установлен)
На Beget Node.js уже должен быть включен через панель. Проверьте версию:
```bash
node -v
```

### 5.4 Установка зависимостей
```bash
npm install --production
```

### 5.5 Сборка проекта
```bash
npm run build
```

### 5.6 Запуск приложения
```bash
npm run start:server
```

## Шаг 6: Настройка автозапуска через PM2

Для того чтобы сайт работал постоянно, установите PM2:

```bash
npm install -g pm2
```

Запустите приложение через PM2:
```bash
pm2 start server.js --name "my-website"
```

Настройте автозапуск:
```bash
pm2 startup
pm2 save
```

## Шаг 7: Проверка работы

1. Откройте ваш домен в браузере
2. Сайт должен быть доступен

## Шаг 8: Настройка SSL (опционально)

1. В панели управления перейдите в "Сайты" → "SSL"
2. Нажмите "Добавить сертификат"
3. Выберите "Бесплатный Let's Encrypt"
4. Следуйте инструкциям

## Альтернативный способ: через Git (рекомендуется)

### 1. Создание репозитория на Beget
1. В панели управления перейдите в "Сайты" → "Git"
2. Нажмите "Создать репозиторий"
3. Выберите тип: "Новый репозиторий"
4. Укажите имя репозитория

### 2. Настройка автодеплоя
1. В настройках репозитория перейдите в "Автодеплой"
2. Добавьте команды для выполнения после деплоя:
```bash
npm install --production
npm run build
pm2 restart my-website
```

### 3. Пуш в репозиторий Beget
В локальном терминале:
```bash
# Добавьте репозиторий Beget как remote
git remote add beget ssh://username@server.beget.com/home/username/git-repos/my-website.git

# Запушьте код
git push beget main
```

## Возможные проблемы и решения

### Проблема: Порт 3000 занят
**Решение:** Измените порт в server.js или используйте переменную окружения PORT.

### Проблема: Ошибка сборки
**Решение:** Проверьте версию Node.js на сервере. Должна быть 18.x или выше.

### Проблема: Изображения не отображаются
**Решение:** Настройте next.config.mjs для работы с внешними доменами изображений.

### Проблема: API маршруты не работают
**Решение:** Убедитесь, что server.js правильно обрабатывает все маршруты.

## Полезные команды PM2

```bash
# Просмотр статуса
pm2 status

# Просмотр логов
pm2 logs my-website

# Перезапуск
pm2 restart my-website

# Остановка
pm2 stop my-website

# Удаление
pm2 delete my-website
```
